React生命周期的一些总结

React生命周期的一些总结

此文档结合React.Component的API食用。

API

Mounting

These methods are called when an instance of a component is being created and inserted into the DOM:

这些方法会在组件创建和插入DOM时触发

  • constructor()
  • componentWillMount()
  • render()
  • componentDidMount()

Updating

An update can be caused by changes to props or state. These methods are called when a component is being re-rendered:

这些方法在组件重新渲染的时候被触发

  • componentWillReceiveProps()
  • shouldComponentUpdate()
  • componentWillUpdate()
  • render()
  • componentDidUpdate()

Unmounting

This method is called when a component is being removed from the DOM:

这些方法会在组件从dom中移除时被触发

  • componentWillUnmount()

阶段性介绍

Before Mount

render（）

执行前检测this.props和this.state并返回一个React元素。render()应该是纯净的。意思是它不应该去更改组件的状态，每次它被引用的时候都会返回相同的结果，它不直接的和浏览器交流。交流的工作交给componentDidMount()

constructor()

组件状态、方法绑定处理的正确地方

componentWillMount()

在组件加载前的钩子函数，组件内容尚且未加载

Mounted

componentDidMount()

此时DOM节点已经加载完毕，此处是一个用来发请求那数据的好地方，这里更新状态会触发重新渲染（render）。

Update

componentWillReceiveProps()

componentWillReceiveProps(nextProps)
这个方法会在新的props或者state被接收到的时候被触发。需要注意的是就算没有新的props这个方法可能也会被触发。（父元素的重渲染导致）所以你要选择是否进行更新你的数据。

shouldComponentUpdate()

shouldComponentUpdate(nextProps, nextState)
本来但凡有state的变化都会发生重渲染，但是这个方法提供给你让你选择是否进行重渲染。首次渲染和forceUpdate()不会触发它。

componentWillUpdate()

componentWillUpdate(nextProps, nextState)
这个方法为更新的到临做准备

componentDidUpdate()

componentDidUpdate(prevProps, prevState)
更新完了之后的方法，这里也是个发请求的好地方。

Unmount

componentWillUnmount()

componentWillUnmount()
组件清除后利用这方法做清理
